入门指南

kAIboard 为什么需要通讯录权限？

kAIboard 读取您的通讯录，将当前聊天对象与其电话号码进行关联。通过识别号码的国际区号（例如 +49 代表德国），kAIboard 自动确定应切换到的语言。联系人的姓名和电话号码不会被永久保存——仅在需要识别语言时临时使用。

kAIboard 为什么需要无障碍服务权限？

此权限使 kAIboard 能够检测您在 WhatsApp、Telegram 或 Signal 等即时通讯应用中正在与哪位联系人聊天。没有此权限，自动语言切换功能将无法工作。kAIboard 仅读取对话标题（即联系人姓名），绝不会读取您的消息内容。

安装后如何设置 kAIboard？

  1. 打开 kAIboard 应用
  2. 在设备的键盘设置中启用 kAIboard 输入法
  3. 将 kAIboard 设为默认键盘
  4. 在系统提示时授予通讯录权限
  5. 启用两项无障碍服务：kAIboard 使用两项服务（一项用于检测聊天中的当前联系人，另一项用于支持非标准界面的应用）。两者都必须启用，自动切换才能正常工作。启动应用时，弹窗会提醒您缺少哪些权限，点击后将直接跳转到设备设置中的权限页面
  6. 在应用设置中启用"通知访问"（用于在通知快捷回复中自动切换语言）
  7. 在应用设置中启用"通知设置"（用于接收使用提醒，例如 AI 翻译的 Token 用量警告）
  8. 开始聊天吧——语言会自动切换！

需要额外下载什么吗？

自动纠错词典已内置于应用中，覆盖全部 13 种语言——使用自动纠错无需下载任何内容。但对于下一词预测功能，您需要为每种想要使用的语言下载预测模型。首次使用某种语言时，您可以在应用内的"离线模型"界面下载模型：点击目标语言旁边的蓝色"下载"按钮。下载完成后，按钮变为红色"移除"，方便您删除不再使用的语言以释放空间。您最常使用的语言会被预加载到内存中，以确保即时切换。

如果您发现某种语言的预测功能不工作，请在"离线模型"界面检查该语言的预测模型是否已下载。

kAIboard 支持哪些语言？

kAIboard 支持 13 种语言，提供完整键盘、自动纠错和预测输入：意大利语、德语、英语、西班牙语、法语、葡萄牙语、荷兰语、波兰语、土耳其语、希腊语、俄语、乌克兰语和格鲁吉亚语。每种语言都有专属的优化键盘布局和内置 Hunspell 词典（已内置于应用中）。如需下一词预测功能，请在"离线模型"界面为每种语言下载对应的 KenLM 模型。

应用的完整界面——包括设置、菜单和所有屏幕文字——已完整翻译为 14 种语言（以上 13 种语言加简体中文）。支持拼音输入的中文键盘即将在未来版本中推出。

功能介绍

自动语言切换是如何工作的？

当您打开一个聊天窗口时，kAIboard 会检测联系人姓名，在通讯录中查找其电话号码并读取国际区号。例如，+39 表示意大利语，+49 表示德语，+33 表示法语。键盘布局会在大约 65 毫秒内自动切换。系统使用多级优先级链：首先检查您是否在该特定应用中为该联系人手动设置了语言，然后检查是否有该联系人的全局设置，接着是通过电话区号自动检测，最后使用设备的系统语言作为后备方案。

您也可以通过两种方式手动设置或更改任意联系人的语言：在应用内的"联系人管理"界面（您可以查看和编辑通讯录中每位联系人关联的语言），或在聊天时直接在键盘上长按空格键（会出现可用语言菜单，您的选择会为该联系人保存）。

如果我有两个同名联系人怎么办？

kAIboard 会弹出一个对话框，让您选择正确的联系人。您的选择会在当前会话中被记住，在键盘服务重启之前不会再次询问。

kAIboard 支持群聊吗？

支持。对于无法检测到单个联系人的群聊，您可以手动指定一种语言。kAIboard 会通过隐私安全的哈希值记住您的选择——群组名称永远不会被保存。

支持哪些即时通讯应用？

超过 120 款应用，包括 WhatsApp、Telegram、Signal、Facebook Messenger、Viber、微信、Discord、Slack 等众多应用——还包括交友应用、网约车应用、外卖应用和电商平台。通知快捷回复同样支持。

关于 Telegram 的说明

Telegram 在内部管理联系人姓名、昵称和数字 ID 的方式与其他即时通讯应用不同。由于 Telegram 的这一特殊行为，当您进入或重新进入与通过通讯录电话号码自动识别的联系人的聊天时，可能偶尔会注意到系统语言的短暂闪现。闪现会很快消失，系统会在之后立即设置正确的语言。对于您已手动分配语言的联系人（通过"联系人管理"界面或长按空格键），语言切换应流畅无延迟，即使在 Telegram 中也是如此。此行为是由 Telegram 向系统传递联系人信息的方式导致的，不是 kAIboard 的问题。在其他即时通讯应用（WhatsApp、Signal、Messenger 等）中，自动和手动联系人的语言切换均应流畅运行。

AI 翻译

AI 翻译是如何工作的？

kAIboard 的 AI 翻译功能具有两个控制级别：

1. 总开关（在应用设置中）：这是主开关。关闭时，所有聊天均无法翻译，且所有单独聊天的翻译设置将被重置。重新开启后，您需要在每个聊天中通过长按回车键重新激活翻译。这是有意为之的设计：防止翻译在被遗忘的聊天中重新激活，在您不知情的情况下消耗 Token。

2. 逐聊天激活（长按回车键）：在总开关开启的状态下，长按回车键为该特定对话激活翻译。按键图标从"↩"（普通回车）变为"⇄"（翻译已激活）。再次长按可关闭该聊天的翻译。该偏好按联系人保存：下次打开同一聊天时，翻译将保持您上次设置的状态（激活或关闭）。

如何翻译：激活翻译后，用您自己的语言正常输入消息。当您在 kAIboard 键盘上按下回车键时，文本会在发送前自动翻译为联系人的语言。然后使用即时通讯应用的发送按钮（文本栏旁边）发送翻译后的消息。翻译仅在按下键盘的回车键时发生——您可以自由输入文字和标点符号，不会受到任何干扰。

翻译需要有效的订阅和网络连接。仅翻译发出的文本（您输入的内容）——kAIboard 绝不会读取、拦截或以任何方式处理收到的消息。这一选择是为了确保最大程度的隐私保护：键盘无法访问接收到的对话内容。

与使用非拉丁语言（俄语、乌克兰语、希腊语、格鲁吉亚语）的联系人聊天时会怎样？

当为使用非拉丁字母语言（俄语、乌克兰语、希腊语、格鲁吉亚语）的联系人激活翻译时，键盘应自动切换到设备系统语言的布局（例如中文），以便您舒适地输入。您用自己的语言输入，翻译会在您按下回车键时将文本转换为联系人的字母和语言。

当为同一联系人关闭翻译时（再次长按回车键），布局应自动恢复为联系人的原生字母（西里尔字母、希腊字母或格鲁吉亚字母）。

注意：此功能已包含在应用中，但尚未进行广泛测试。在某些情况下，布局自动切换可能无法按预期工作。如遇问题，请通过 support@kaiboard.eu 与我们联系。

翻译安全吗？我的消息会被保存吗？

文本通过 HTTPS 加密连接安全传输到 Cloudflare 中转服务器，再转发至 OpenAI API 进行翻译。文本绝不会保存在任何服务器上——仅作短暂中转后立即返回。OpenAI API 密钥绝不会在您的设备上暴露。

关于 OpenAI 的透明声明：kAIboard 使用 OpenAI 作为外部翻译服务提供商。OpenAI 公开声明，通过 API 发送的数据不会用于训练其模型。欧盟外数据传输的标准合同条款已生效。但是，我们目前尚未与 OpenAI 签订正式的数据处理协议（DPA）。我们建议您避免翻译高度敏感的信息（健康数据、财务数据、法律数据）。详情请参阅 OpenAI API 隐私政策。

预测输入与自动纠错

kAIboard 的预测输入与其他键盘相比如何？

kAIboard 的预测引擎基于坚实的技术基础：统计语言模型、完整的词典，以及随着您每次输入不断优化的个性化学习系统——所有处理均在设备本地完成，无需依赖云端。不过，kAIboard 由一位独立开发者开发。大型科技公司拥有由数百名工程师组成的专业团队，使用数十亿数据训练神经网络模型。kAIboard 的预测功能扎实可靠，会随着使用而显著提升，但我不会声称能达到那样的规模。真正让 kAIboard 与众不同的是其他键盘都做不到的功能：在您输入第一个字符之前，就知道您需要哪种语言。

自动纠错是如何工作的？

kAIboard 的自动纠错采用多因素系统，分析输入单词与词典候选词之间的编辑距离、句子上下文（前面的单词）、单词的实际使用频率、键盘上按键的相邻关系以及您个人的输入习惯。系统支持纠正字母调换（顺序颠倒的字母）、缺失的重复字母、误按的相邻键和缺失的重音符号——覆盖所有 13 种支持语言。

自动纠错采用保守策略：在建议栏中显示纠正建议，仅在系统足够确信时才自动应用。没有什么比键盘擅自覆盖您真正想输入的内容更令人沮丧的了。控制权始终在您手中。

kAIboard 会从我的输入习惯中学习吗？

会的。学习系统会记录您最常使用的单词、最常见的词语组合以及您输入的专有名词。建议会随着时间推移而改善，通过智能衰减系统保持近期习惯比旧习惯更具相关性。学习仅在您点击建议栏中的建议时发生——自动纠正不会被记录，以防止纠错错误被当作习惯学习。所有学习数据均保存在您的设备上，可随时删除。

隐私与数据

kAIboard 收集哪些数据？

kAIboard 采用本地优先的设计理念。联系人检测、自动纠错、预测输入和语言偏好设置均在您的设备本地完成。唯一发送到外部的数据是您主动选择翻译的文本（通过安全的 HTTPS 加密连接传输），且不会保存在任何服务器上。无数据分析、无用户追踪、无广告标识符。

kAIboard 符合 GDPR 规定吗？

是的。kAIboard 在隐私设置中提供四个选项：以 JSON 格式导出您的数据、仅删除学习数据、删除所有本地数据、或删除所有云端数据。云端数据（订阅信息）会在 90 至 365 天不活跃后自动删除。

崩溃报告完全可选：您可以随时在设置中启用或禁用。首次启用时会显示确认对话框。崩溃报告仅包含技术错误信息和设备元数据——绝不包含输入的文本、联系人姓名或其他个人数据。

应用中显示的管理员日志是什么？

为确保最大透明度，kAIboard 在每次从服务器端对您的账户进行更改时会自动记录日志——例如，因争议或技术问题而进行的 Token 补偿。这些日志在应用内的相应部分可见，以便您随时查看是否以及何时对您的账户执行了操作。如果没有日志，则表示从未进行过任何服务器端更改。管理员日志会根据数据保留政策自动删除，您也可以通过隐私部分的"删除云端数据"按钮手动删除。

我可以删除所有数据吗？

可以。进入"设置"→"隐私"，使用相应按钮操作。您可以分别导出 JSON 格式的数据、删除学习数据、删除所有本地数据或删除云端数据。卸载应用会自动删除所有本地数据。

价格与订阅

kAIboard 多少钱？

kAIboard 为一次性购买，价格 €2.39（含税）。包含所有核心功能：自动语言切换、智能纠错、下一词预测、Emoji 选择器、联系人语言管理和本地备份。

订阅增加了哪些功能？

可选的月度订阅解锁两项高级功能：AI 翻译（由 OpenAI 提供支持）和 Google Drive 备份。根据每月翻译用量，提供三种方案：Plus（€3.49/月 — 约 2,130 次翻译）、Pro（€5.99/月 — 约 4,200 次翻译）和 Max（€9.49/月 — 约 8,530 次翻译）。

订阅可以在多台设备上使用吗？

可以。您的 Token 余额会在使用同一 Google Play 账户的所有设备间同步。每月 Token 限额为共享额度——例如，Plus 方案的 250,000 个 Token 将在您的所有设备间共享使用。

取消订阅后重新安装应用会怎样？

订阅有效（未取消）时：如果您卸载并重新安装应用，或者使用同一 Google 账户在新设备或多台设备上安装应用，订阅和 Token 余额将通过 Google Play 和 Firebase 自动同步。您无需执行任何操作——首次启动时系统将自动恢复。

订阅已取消（宽限期内）时：如果您在同一设备上重新安装应用，系统将通过 Firebase 找到您的订阅和 Token 余额，并自动恢复，即使在宽限期内也是如此。在使用同一账户且之前已安装过该应用的其他设备上，如果该设备已在系统中注册，恢复也应自动完成。

重要提示：如果在宽限期内，您在一台从未使用过您的账户安装 kAIboard 的全新设备上安装应用，由于该新设备尚未在系统中注册，出于安全原因，剩余天数和 Token 余额可能无法恢复。同样，如果您在宽限期内从应用中删除云端数据，删除将立即生效，届时您将永久失去剩余有效期和 Token 余额。

备份是如何工作的？

kAIboard 提供两种备份方式。所有用户均可创建设置和学习数据的本地备份。订阅用户还可以备份到个人 Google Drive。无论哪种方式，翻译 Token 余额始终从云端同步——不会保存在备份文件中，以防止数据重复。

故障排除

键盘没有自动切换语言。

请确认通讯录权限和无障碍服务均已启用。检查该联系人在您的通讯录中是否有带国际区号的电话号码。

翻译功能不工作。

请确认您有有效的订阅且有可用的 Token。检查应用设置中的翻译总开关是否已开启。确认您已通过长按回车键为特定聊天激活了翻译（图标应显示 ⇄）。翻译功能还需要网络连接。

回车键图标没有显示翻译箭头。

请检查应用设置中的翻译总开关是否已开启。如果已开启，请尝试退出聊天并重新进入。图标仅在您通过长按回车键激活了翻译的聊天中才会改变。
